# A-1 데이터 카탈로그 가이드

# Executive Summary

데이터 카탈로그는 단순히 데이터의 위치를 기록하는 목록이 아니라, 계열사 내 데이터 자산을 체계적으로 식별·관리·재사용하기 위한 AI-ready Data 체계의 핵심 기반이다.

AI 과제 수행 시 가장 많은 시간이 소요되는 활동 중 하나는 필요한 데이터의 존재 여부, 위치, 오너, 접근 방법을 파악하는 과정이다. 데이터 카탈로그는 이러한 탐색 비용을 최소화하여 AI 과제 수행 속도를 높이고, 기존에 축적된 데이터 자산을 재사용할 수 있도록 지원한다.

또한 데이터 카탈로그는 데이터 자체뿐 아니라 AI 과제 수행 과정에서 생성되는 전처리 데이터, 평가 데이터, Prompt, Ontology 등의 결과물을 지속적으로 축적하기 위한 기반 체계로 활용되어야 한다. 이를 통해 개별 과제 단위의 산출물이 아닌 그룹 차원의 AI 자산을 축적할 수 있다.

본 가이드는 데이터 카탈로그 구축 방법뿐 아니라 운영 체계, AI 과제 연계 방안, KPI, 확산 전략을 포함하여 정의한다.

# 1. 데이터 카탈로그 개요

## 1.1 데이터 카탈로그의 정의

데이터 카탈로그는 조직이 보유한 데이터 자산의 존재 여부, 위치, 오너, 접근 경로 및 활용 정보를 통합 관리하는 체계이다.

데이터 자체를 저장하거나 이동하는 시스템이 아니라, 데이터 자산을 발견하고 활용할 수 있도록 지원하는 데이터 자산 관리 계층으로 이해할 수 있다.

데이터 카탈로그의 핵심 역할은 다음과 같다.

- 데이터 자산 존재 여부 식별
- 데이터 위치 및 접근 경로 제공
- 데이터 오너 및 관리 조직 식별
- 데이터 탐색 및 검색 지원
- 데이터 재사용 촉진
- AI 활용 대상 데이터 식별

데이터 카탈로그는 데이터의 의미나 품질 자체를 정의하지 않는다.

| 질문 | 담당 영역 |
|--------|--------|
| 데이터가 어디에 존재하는가 | 데이터 카탈로그 |
| 데이터 필드와 구조는 무엇인가 | 메타데이터 |
| 업무 용어 의미는 무엇인가 | 비즈니스 Glossary |
| 데이터가 어떻게 생성되었는가 | Lineage |
| 데이터를 AI에 활용할 수 있는가 | 데이터 품질 관리 |

## 1.2 데이터 카탈로그 구축 목적

데이터 카탈로그 구축 목적은 단순한 데이터 탐색 지원을 넘어 계열사 AI 자산화를 위한 기반 체계를 구축하는 데 있다.

### 목적 1. 데이터 탐색 시간 최소화

데이터 규모와 시스템 수가 증가할수록 데이터를 찾는 데 소요되는 시간은 증가한다.

데이터 카탈로그는 데이터 위치, 오너, 접근 경로를 표준화하여 데이터 탐색 시간을 최소화한다.

### 목적 2. 데이터 재사용 확대

동일한 데이터를 반복적으로 수집하거나 가공하는 비효율을 줄이고, 기존 데이터 자산의 재사용을 촉진한다.

이를 통해 AI 과제 수행 시 데이터 확보 비용을 절감할 수 있다.

### 목적 3. AI 과제 수행 가속화

AI 과제 초기 단계에서 반복적으로 발생하는 데이터 탐색, 데이터 확보, 오너 확인 과정을 표준화하여 실제 분석과 모델 개발에 집중할 수 있도록 지원한다.

### 목적 4. AI 자산 지속 축적

데이터 카탈로그는 원천 데이터뿐 아니라 전처리 데이터, 평가 데이터, Prompt, Ontology 등 AI 과제 수행 과정에서 생성되는 자산을 지속적으로 축적하기 위한 기반 체계로 활용한다.

### 목적 5. 그룹 표준 체계 구축

계열사별 상이한 데이터 관리 방식을 표준화하고 향후 그룹 차원의 AI 활용 및 데이터 자산 연계를 위한 공통 체계를 마련한다.

## 1.4 AI-ready Data 체계 내 역할

AI-ready Data 체계에서 데이터 카탈로그는 데이터 활용의 시작점 역할을 수행한다.

모든 데이터 자산은 카탈로그에 등록되어야 하며, 등록된 자산만이 이후 데이터 품질 관리, 전처리, 주석, AI 학습 등의 단계로 연결될 수 있다.

데이터 카탈로그는 AI-ready Data 체계 내에서 다음 역할을 수행한다.

- 데이터 자산 식별
- 데이터 자산 탐색
- 데이터 자산 재사용
- AI 활용 데이터 관리
- AI 자산 축적 기반 제공

# 2. 데이터 카탈로그 필요성 및 기대효과

## 2.1 AI 과제 수행 시 주요 문제점

AI 과제 수행 시 가장 빈번하게 발생하는 문제는 필요한 데이터를 찾는 과정에서 발생한다.

대표적인 문제는 다음과 같다.

### 데이터 존재 여부 확인 불가

데이터가 존재하는지 여부를 특정 담당자의 경험에 의존한다.

### 데이터 위치 파편화

ERP, MES, QMS, LIMS, SharePoint, 파일 서버 등 다양한 시스템에 데이터가 분산되어 있다.

### 오너 및 접근 절차 불명확

데이터 접근을 위해 누구에게 요청해야 하는지 확인하기 어렵다.

### 데이터 재사용 부족

기존에 정제된 데이터가 존재함에도 이를 알지 못해 동일한 작업을 반복 수행한다.

### AI 과제 착수 지연

데이터 확보 단계에 과도한 시간이 소요되어 AI 과제 수행 기간이 증가한다.

---

## 2.2 데이터 카탈로그 구축 필요성

데이터 자산 수가 증가할수록 데이터 탐색 비용은 증가한다.

특히 제조 기업은 ERP, MES, QMS, LIMS, IoT, SharePoint 등 다양한 시스템을 운영하고 있기 때문에 데이터 자산의 위치를 체계적으로 관리하지 않으면 AI 활용이 어려워진다.

또한 AI는 사람처럼 담당자에게 질문할 수 없다.

AI Agent가 데이터를 탐색하기 위해서는 데이터 위치와 활용 정보를 기계가 이해할 수 있는 형태로 제공해야 하며, 데이터 카탈로그는 이러한 역할을 수행한다.

데이터 카탈로그 구축을 통해 다음과 같은 효과를 기대할 수 있다.

| 활용 | 데이터 재사용 확대 |

| AI | AI 과제 수행 가속화 |
| 자산화 | AI 자산 지속 축적 |
| 확산 | 계열사 표준 체계 구축 |

---

## 2.4 데이터 카탈로그의 핵심 기능

데이터 카탈로그는 다음 네 가지 기능을 중심으로 구성된다.

### 1. 등록 항목 관리

데이터 자산의 기본 정보를 등록하고 관리한다.

### 2. 탐색 체계 제공

사용자와 AI가 데이터를 쉽게 찾을 수 있도록 검색 및 분류 체계를 제공한다.

### 3. 수집 및 연계

다양한 시스템의 데이터 자산 정보를 수집하고 통합한다.

### 4. 최신성 유지

데이터 위치, 오너, 접근 경로 등의 정보를 최신 상태로 유지한다.

# 5. 데이터 현황 조사 및 등록 대상 선정

## 5.1 기본 원칙

모든 데이터를 등록하는 것을 목표로 하지 않는다.

AI 활용 가능성, 업무 중요도, 재사용성을 고려하여 우선순위를 정의한다.

---

## 5.2 등록 대상 선정 기준

### AI 활용도

AI 학습, 추론, Agent 활용 가능 여부

---

### 업무 중요도

업무 수행에 미치는 영향 수준

---

### 재사용성

여러 조직 또는 과제에서 반복 활용 가능 여부

---

### 구축 난이도

자동 수집 가능 여부 및 구축 비용

---

## 5.3 우선순위 산정 방식

등록 우선순위는 아래 기준으로 산정한다.

총점 =

AI 활용도 (30)

+ 재사용성 (25)

+ 업무 중요도 (25)

+ 구축 난이도 (20)

---

### AI 활용도

| 기준 | 점수 |
|------------|------------|
| 현재 AI 활용 중 | 30 |
| 활용 예정 | 20 |
| 활용 계획 없음 | 10 |

---

### 재사용성

# 8. 데이터 카탈로그 운영

## 8.1 운영 원칙

데이터 카탈로그는 구축보다 운영이 중요하다.

초기 등록 이후 지속적인 갱신과 활용이 이루어지지 않으면 데이터 카탈로그는 빠르게 활용도가 낮아진다.

운영 시 아래 원칙을 적용한다.

### 원칙 1. 데이터 오너 책임 기반 운영

모든 데이터 자산은 오너를 지정한다.

오너는 다음 사항에 책임을 가진다.

- 등록
- 변경
- 최신성 유지
- 접근 승인

---

### 원칙 2. 최신성 유지

데이터 위치, 오너, 접근 경로 정보는 실제 환경과 일치해야 한다.

아래 이벤트 발생 시 갱신한다.

- 신규 데이터 생성
- 데이터 폐기
- 위치 변경
- 오너 변경

---

### 원칙 3. 활용 중심 운영

단순 등록 건수보다 실제 활용 여부를 중요하게 관리한다.

# 12. Roadmap

## 12.1 추진 방향

데이터 카탈로그는 단순 구축 프로젝트가 아니라 AI 자산 축적 체계로 발전해야 한다.

Roadmap은

구축

↓

운영

↓

확산

↓

자동화

순으로 발전한다.

---

## Phase 1. 표준 체계 수립

목표

데이터 카탈로그 운영 기반 마련

주요 과제

- 표준 정의
- 등록 기준 정의
- KPI 정의

주요 산출물

- 운영 가이드
- 등록 표준
- KPI 체계

---

## Phase 2. 데이터 카탈로그 구축

목표

핵심 데이터 등록

주요 과제

- 시스템 연계
- 데이터 등록
- 운영 체계 구축

주요 산출물

- 카탈로그 시스템
- Wave1 데이터

---

## Phase 3. AI 과제 연계

목표

AI 과제 결과물 축적

주요 과제

- AI 자산 등록
- Prompt 등록
- Ontology 등록

주요 산출물

- AI 자산 Repository

---

## Phase 4. 계열사 확산

목표

그룹 공통 체계 구축

주요 과제

- 계열사 확대
- KPI 운영
- 성숙도 관리

주요 산출물

- 그룹 통합 현황 Dashboard

---

## Phase 5. AI 기반 자동화

목표

AI Agent 활용

주요 과제

- 자동 탐색
- 자동 추천
- 자동 등록

주요 산출물

- AI Agent 연계 체계

---

## 12.2 최종 목표

최종적으로 데이터 카탈로그는 데이터 목록 관리 시스템이 아니라 그룹 차원의 AI 자산 관리 플랫폼으로 발전한다.

```text
2026
↓
데이터 등록

2027
↓
AI 자산 축적

2028
↓
계열사 확산

2029+
↓
AI Agent 기반 자동 활용
```

이를 통해 신규 AI 과제 수행 시 기존 자산 활용 비율을 지속적으로 확대한다.

목표

- 초기 : 신규 생성 중심
- 중기 : 50% 재사용
- 장기 : 70~90% 재사용
